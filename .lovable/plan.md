
用户的真实需求是：截图标注时能清晰看到要检测的部位，希望通过颜色差异突出检测部分。

这其实不是"改产品颜色"，而是"高亮/标记检测部位"。两种方向：
1. 在 3D 截图前给整个产品换色（让检测点的标注更醒目）
2. 在 3D 视图里直接点选 mesh 高亮（更精准）

考虑到当前流程是「3D 截图 → 进入标注模式画框」，最实用的做法是**让用户在截图前调整产品外观**，使后续标注框/检测点在截图上更突出。

最简单有效的方案：加个"产品外观"控制条，包含底色切换 + 单色染色 + 线框模式三选一，让用户根据产品类型挑一个对比度最高的呈现方式。

## 方案：3D 截图前的"对比度增强"控制

在 `ProductViewerCanvas.tsx` 工具栏新增一组控制（截图按钮左侧），在 `Product3DViewer.tsx` 内部 `Model` 组件里实现对应的渲染效果。

### 控制项（4 个，一行排开）

1. **背景色切换**：浅灰（默认）/ 纯白 / 纯黑 / 中性灰
   - 深色产品配白底，浅色产品配黑底，对比立现
2. **产品染色**：原色（默认）/ 取色器
   - 选了颜色后整个模型染成单色，保留明暗细节（用 `material.color` 而非贴图）
   - 配一个"恢复原色"按钮
3. **显示模式**：实体（默认）/ 半透明 / 线框
   - 半透明可看到内部结构，便于标注内部检测点
   - 线框模式凸显边缘和孔位
4. **检测点高亮辅助**：开关一个"网格叠加"
   - 在产品表面叠一层细网格，标注时容易对齐

### 实现要点

- 状态全部放在 `Product3DViewer` 内部，不持久化（每次进入重置）
- `Model` 组件接收 `tintColor`、`renderMode`、`showGrid` 三个 prop
- 渲染时 clone 材质再修改，避免污染缓存的 GLTF（GLB 缓存 isolation 已是项目核心约束）
- 截图通过现有 `captureWebGLFrameToDataUrl` 流程，所见即所得
- 切换时保留相机位置和视角，不打断用户操作

### 影响范围

- `src/components/product/Product3DViewer.tsx`：加控件 UI + Model 组件接收新 prop
- `src/components/canvas/ProductViewerCanvas.tsx`：可选，若希望工具栏更紧凑可在此聚合
- 不影响：截图链路、标注、PPT/DOCX 导出、其他模块

### 不做的事

- 不做按部件分别染色（复杂、用户没明确需要）
- 不做材质 PBR 调节（metalness/roughness，用处不大）
- 不持久化用户选择（避免污染 zustand store，符合项目约束）
